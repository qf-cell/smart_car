# 双车协同控制系统 —— 主控车视觉开发者指南

## 系统概览

```
┌──────────┐     uart6(9600)      ┌──────────────┐   wireless_uart(115200)   ┌──────────┐
│  上位机   │ ──────────────────→  │   主控车      │ ────────────────────────→ │  跟随车   │
│ (视觉/UI) │ ←────────────────── │  (RT1021)    │                           │ (RT1021) │
└──────────┘    位置/状态回传      └──────────────┘                           └─────┬────┘
                                                                                   │
                                                       摄像头 uart6(115200) ←──────┘
                                                        @VW#vx#vy#w!  视觉反馈

两车车头同向，跟随车在主车左侧 20cm 处。
```

- **主控车**：接收上位机指令，通过无线串口向跟随车发送**前馈**（自身目标速度/角速度）
- **跟随车**：融合前馈（来自主车）和视觉反馈（来自摄像头），跟踪主车运动
- **你的工作**：视觉模块（上位机侧或主车侧）通过 uart6 向主控车发送控制指令，主控车自身同时自动向跟随车转发前馈数据

## 主控车硬件连接

| 串口            | 波特率  | 用途                               |
|-----------------|---------|------------------------------------|
| uart6  (UART5) | 9600    | **← 接收上位机/视觉指令**           |
| wireless_uart   | 115200  | **→ 向跟随车发送前馈** (`@VW` 帧)   |

> `hardware_init.py` 中：`uart6 = UART(5)`，`wireless_uart = UART(2)`。

## uart6 控制协议（上位机 → 主控车）

### 帧格式

所有指令统一格式：

```
@<命令类型>#<参数1>#<参数2>...!
```

帧以 `@` 开头、`!` 结尾，参数以 `#` 分隔。**9600 波特率，8N1**。

### 指令表

| 指令        | 格式               | 说明                              | 响应                  |
|------------|---------------------|-----------------------------------|-----------------------|
| 设置位置    | `@P#x#y!`           | 设置世界坐标目标点，主控车自动导航  | 到达后回发 `@P#px#py!`|
| 设置速度    | `@V#vx#vy!`         | 直接设置车体速度，关闭导航         | —                     |
| 设置角度    | `@A#angle!`         | 设置目标 yaw 角（度）             | —                     |
| 设置坐标    | `@S#x#y!`           | 重置当前位置为指定世界坐标         | `@P#px#py!`           |
| 获取位置    | `@G!`               | 查询当前位置                       | `@P#px#py!`           |

### `@P` — 设置目标位置（导航模式）

```
@P#x#y!
```

| 参数 | 含义         | 单位 | 示例    |
|------|-------------|------|---------|
| x    | 世界坐标 x   | m    | `1.5`   |
| y    | 世界坐标 y   | m    | `2.0`   |

指令示例：`@P#1.5#2.0!`

主控车收到后启用位置 PID 导航，自动驶向目标点。到达目标（位置误差 < 0.01m 持续 100ms）后关闭导航、回发确认帧 `@P#px#py!`。

### `@V` — 直接设置速度

```
@V#vx#vy!
```

| 参数 | 含义         | 单位  | 方向                  |
|------|-------------|-------|-----------------------|
| vx   | x 轴速度     | m/s   | +x 为车头前方          |
| vy   | y 轴速度     | m/s   | +y 为车体左侧          |

指令示例：`@V#0.2#0.0!`（以 0.2m/s 前进）

> 发送 `@V` 后导航自动关闭。适用于视觉模块实时计算速度的连续控制场景。

### `@A` — 设置目标角度

```
@A#angle!
```

| 参数   | 含义           | 单位 | 方向                      |
|--------|---------------|------|---------------------------|
| angle  | 目标 yaw 角    | 度   | 逆时针为正（左转为正）     |

指令示例：`@A#45.0!`

主控车内置角度 PID，收到后自动闭环旋转到目标角度。

### `@S` — 设置当前坐标

```
@S#x#y!
```

不触发运动，仅将主控车当前的里程计位置重置为指定坐标。回发 `@P#px#py!` 确认。

### `@G` — 获取当前位置

```
@G!
```

主控车通过 uart6 回发 `@P#px#py!`，包含当前的导航里程计坐标。

## 坐标系

```
        +y
         ↑
         │  (车体左侧)
    ┌────┴────┐
    │         │
    │  主控车  │ → +x (车头前方)
    │         │
    └─────────┘
```

- **车体坐标系**：+x 为车头前方，+y 为车体左侧
- **世界坐标系**：初始化时主控车位置为 (0, 0)，方向由 IMU yaw 角定义
- **角速度**：逆时针为正（从上往下看，左转为正）
- **角度**：逆时针为正（`car.angle`，来自 IMU yaw）

## 前馈：主控车 → 跟随车

主控车在主循环中**自动**通过无线串口向跟随车发送前馈帧，无需视觉模块干预。

### 前馈帧格式

```
@VW#vx#vy#w!
```

| 参数 | 含义           | 说明                                         |
|------|---------------|----------------------------------------------|
| vx   | 前馈 x 轴速度  | 来自 `car.target_vx`                         |
| vy   | 前馈 y 轴速度  | 包含**环绕补偿**（跟随车在左侧 20cm）          |
| w    | 前馈角速度     | `|w| < 0.2` 时发送为 0（微调死区），单位 rad/s |

### 环绕补偿

跟随车在主车左侧 20cm，主车旋转时跟随车绕主车做圆周运动。补偿公式：

```python
vy_ff = car.target_vy + car.target_w * 0.2  # 0.2m 偏移
```

### 发送频率

前馈帧每 **10ms** 发送一次（主循环 5ms 周期中每 2 次发 1 次）。

## 主控车关键对象与属性

### `car` (Car 实例)

| 属性             | 含义                   | 单位   |
|------------------|------------------------|--------|
| `car.target_vx`  | 目标 x 轴速度           | m/s    |
| `car.target_vy`  | 目标 y 轴速度           | m/s    |
| `car.target_w`   | 目标角速度（角度PID输出）| rad/s  |
| `car.target_angle`| 目标 yaw 角            | 度     |
| `car.angle`      | 当前 yaw 角（IMU）      | 度     |
| `car.fused_vx`   | 融合后的实际 x 轴速度    | m/s    |
| `car.fused_vy`   | 融合后的实际 y 轴速度    | m/s    |

### `navigation` (Navigation 实例)

| 属性                     | 含义             | 单位 |
|--------------------------|------------------|------|
| `navigation.px`          | 世界坐标 x       | m    |
| `navigation.py`          | 世界坐标 y       | m    |
| `navigation.enable_navigation` | 导航使能标志 | —    |

## 控制周期

```
主循环 (5ms)
  └─ uart_control()          # 读取 uart6 指令
  └─ uart_send_feedforward() # 每10ms向跟随车发前馈

PIT 中断 (10ms)
  └─ IMU 姿态解算
  └─ 角度 PID 控制
  └─ 全向轮速度分配
  └─ 编码器 + IMU 速度融合
  └─ 位置导航（若启用）
  └─ 电机 PID 闭环
```

## 快速上手

1. **物理连接**：上位机/视觉模块 UART 连接到 RT1021 开发板的 uart6（`UART(5)` 引脚，参考逐飞学习板引脚定义）
2. **串口配置**：**9600 波特率，8 数据位，1 停止位，无校验**
3. **发送指令**：按 `@命令#参数!` 格式发送，主控车立即响应
4. **读取回传**：监听同一串口，主控车会回发 `@P#px#py!`
5. **观察 LCD**：主控车 LCD 显示实时速度和位置，用于快速确认指令是否生效

### 典型流程

```
视觉模块 → @S#0#0!                         (设原点)
视觉模块 → @P#1.0#0.5!                     (导航到目标点)
主控车   → @P#1.00#0.50!                   (到达确认)

视觉模块 → @V#0.15#0.0!                    (改为速度直控)
视觉模块 → @V#0.15#0.02!                   (微调横向)
```

### 常见问题

- **指令无响应**：检查波特率是否为 9600；帧格式是否正确（`@` 开头 `!` 结尾）
- **导航不生效**：确认 IMU 校准已完成（`imu660RA.init_flag == True`，约需 1s）
- **主控车偏航**：IMU 的 yaw 角是纯积分值，长时间有漂移；可用 `@S` 重置坐标或用 `@A` 闭环角度
- **跟随车不跟踪**：检查无线串口连接和波特率（115200）；观察前馈帧是否发出
