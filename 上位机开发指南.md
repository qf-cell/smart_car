# 主车（主控车）上位机开发指南

## 1. 串口连接

| 项目 | 参数 |
|------|------|
| 串口 | UART6（主车上的物理接口） |
| 波特率 | 9600 |
| 数据位 | 8 |
| 停止位 | 1 |
| 校验 | 无 |

上位机通过 UART6 向主车发送控制指令，主车也会通过该串口返回位置信息。

## 2. 通信帧格式

所有指令统一格式：

```text
@命令字#参数1#参数2#...!
```

- 帧头：`@`
- 帧尾：`!`
- 参数之间用 `#` 分隔
- 参数均为**字符串形式**，主车端会 `float()` 转换

## 3. 指令列表

### 3.1 设置角度 — `@A#angle!`

让主车转到指定角度。

```text
@A#-90!     # 车头朝右
@A#0!       # 车头朝前（上）
@A#90!      # 车头朝左
```

注意：主车推物体时只识别 `-90°`、`0°`、`90°` 三个方向。

### 3.2 设置速度 — `@V#vx#vy!`

直接设置车体速度。

```text
@V#0.0#0.5!     # 向前（vy 方向）0.5m/s
@V#0.3#0.0!     # 向右（vx 方向）0.3m/s
@V#0.0#0.0!     # 停止
```

注意：本车主车坐标系约定 `vy` 为**前进方向**，`vx` 为**侧向**。

### 3.3 设置位置 — `@P#x#y!` 或 `@P#x#y#v!`

让主车导航到指定世界坐标位置。

```text
@P#1.5#0.8!        # 目标位置 (1.5, 0.8)，速度默认 0.3m/s
@P#1.5#0.8#0.5!    # 目标位置 (1.5, 0.8)，速度 0.5m/s
```

### 3.4 获取位置 — `@G!`

主车返回当前位置：

```text
@P#1.23#0.67!
```

### 3.5 设置当前坐标 — `@S#x#y!`

把当前位置强制设为指定坐标，用于校准或重定位。

```text
@S#0.0#0.0!
```

### 3.6 相对移动 — `@D#dx#dy!` 或 `@D#dx#dy#v!`

在当前位置上累加移动。

```text
@D#0.5#0.0!        # 向右移动 0.5m
@D#0.0#0.3#0.4!    # 向前移动 0.3m，速度 0.4m/s
```

### 3.7 开始推物体 — `@PUSH!`

启动推物体流程，带 TOF 避障。

**前提条件**：主车必须先转到 `-90°`、`0°` 或 `90°` 之一，否则 `push_to_edge()` 不会执行。

```text
@A#0!      # 先转到朝前
@PUSH!     # 开始向前推物体
```

### 3.8 停止推物体 — `@PHSP!`

立即停止推物体，并停车。

```text
@PHSP!
```

## 4. 坐标系说明

| 轴 | 含义 |
|---|---|
| `+x` | 朝右 |
| `+y` | 朝前（上） |
| `navigation.px` | 主车在世界坐标系中的 x 位置 |
| `navigation.py` | 主车在世界坐标系中的 y 位置 |

场地边界参数（在主车代码中硬编码）：

| 边界 | 坐标 |
|---|---|
| 右边界 `right_edge_x` | 3.2 m |
| 左边界 `left_edge_x` | 0.0 m |
| 上边界 `top_edge_y` | 2.4 m |
| 下边界 `bottom_edge_y` | 0.0 m |

## 5. 推物体流程

### 5.1 阶段说明

`push_to_edge()` 根据当前车头方向和位置分三个阶段：

| 阶段 | 触发条件 | 行为 |
|---|---|---|
| 快速推进 | 离边缘 > 1.1m（左右）或 > 0.7m（前） | 以 0.5m/s 直线前进 |
| 避障推进 | 离边缘 0.2~1.1m（左右）或 0.2~0.7m（前） | 以 0.3m/s 前进，TOF 扫描避障 |
| 边缘停止 | 离边缘 < 0.2m | TOF 指向前方，根据距离阈值停车 |

### 5.2 边缘停止距离

当前代码中停止距离为：

```python
right_distance_to_area_edge = 500    # mm
left_distance_to_area_edge = 500     # mm
top_distance_to_area_edge = 500      # mm
bottom_distance_to_area_edge = 500   # mm
```

比赛时需要根据实际场地和物体尺寸单独测量调整。

### 5.3 推边成功返回值

当主车到达边缘并停止时，会返回：

```text
@P#x#y!
```

表示当前位置已校准到边缘。

## 6. 避障说明

- 只有在**避障推进阶段**才会启用 TOF 扫描。
- TOF 扫描范围：舵机 60°~140°，覆盖车体前方约 30cm 宽区域。
- 扫描数据分为 20 个 zone，每个 zone 宽 15mm。
- 避障判断：左右半区（各 10 个 zone）分别取有效距离平均值，哪边更靠近障碍物，车就向另一边偏移。
- 返回速度：偏移量 ±0.3m/s，叠加在前进速度 0.3m/s 上。

## 7. 典型控制时序

### 7.1 推物体到前边界

```text
@S#0.5#0.5!      # 校准当前位置
@A#0!            # 转到朝前
@PUSH!           # 开始推
（等待主车返回 @P#x#y!）
```

### 7.2 紧急停止

```text
@PHSP!
```

### 7.3 平移到指定位置

```text
@P#1.0#0.5!      # 导航到 (1.0, 0.5)
（等待主车到达并停止）
```

## 8. 注意事项

1. **推物体前必须先转方向**：`@PUSH!` 只在 `-90°`、`0°`、`90°` 附近有效。
2. **PHSP 会立即停车**：收到 `@PHSP!` 后 `push_flag` 清 0，同时速度设为 0。
3. **位置校准**：推边成功后主车会把位置强制设为边缘坐标，用于后续重定位。
4. **TOF 调试输出**：主车会通过无线串口每秒 10 次输出 zone 数据，可用于调试，但控制指令走 UART6。
5. **边缘距离阈值**：500mm 是默认值，正式比赛前务必实测。
