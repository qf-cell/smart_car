# main.py 逻辑说明文档

## 1. 总体架构

```
初始化 (传感器/串口/熊模型预热)
  ↓
┌─────────────────────────────────────────────────────────────┐
│  while True                                                 │
│                                                             │
│  ╔══════════ 核心扫描循环 ══════════╗                       │
│  ║  step=1  网球扫描  (0°)    X:110→210  ║                  │
│  ║    ↓                                  ║                  │
│  ║  step=2  沙包扫描  (−90°)  Y:170→70   ║  ← 一个循环      │
│  ║    ↓                      + X补扫     ║                  │
│  ║  step=3  小熊扫描  (0°)    Y:70→170   ║                  │
│  ╚══════════════════════════════════════╝                   │
│    ↓                                                        │
│  [循环结束: 检查计数]                                        │
│    │                                                        │
│    ├─ 计数全零? ──YES──→ 返回起点(25,−25) → 信标重定位       │
│    │                      ↓ 重置计数 → 回到核心扫描循环      │
│    │                                                        │
│    └─ 仍有剩余? ──→ 回到核心扫描循环 (step=1)               │
│                      各步骤自动跳过计数=0 的物体              │
└─────────────────────────────────────────────────────────────┘
```

> steps 1-3 构成一个**核心扫描循环**（依次处理网球→沙包→小熊）。step 3 末尾进行**计数检查**：全零则回家+重定位+重置，否则回到 step 1 开始下一轮。各步骤头部自动跳过计数为 0 的物体。

## 2. 坐标系与朝向

| 朝向角度 | 世界方向 | 扫描轴向 |
|---------|---------|---------|
| `0°` | Y↑ (正前方) | 沿 X 轴扫描 |
| `−90°` | X→ (右) | 沿 Y 轴扫描 |
| `90°` | X← (左) | — |
| `180°` | Y↓ (后方) | — |

- 角度范围：`(−180, 180]`，逆时针为正
- 场地尺寸：`320 × 240 cm`
- 起点：`(40, −12)`
- 回家点：`(25, −25)`

## 3. 可配置参数

```python
# 物体个数
TENNIS_TOTAL   = 1    # 网球总个数
EARTHBAG_TOTAL = 1    # 沙包总个数
BEAR_TOTAL     = 1    # 小熊总个数

# 场地
FIELD_WIDTH  = 320    # cm
FIELD_HEIGHT = 240    # cm

# 抵近
CENTER_TOLERANCE_PX = 30   # 横向居中容差 (像素)
OFFSET_CM           = 10   # 推送前偏移量 (cm)
SCAN_TIMEOUT_FRAMES = 300  # 扫描超时 (~10s @30fps)

# 小熊专用
BEAR_SCAN_FRAMES = 300     # 小熊扫描超时帧数
```

## 4. 初始化流程 (`main()`)

```
1. init_bear()          预加载熊 tflite 模型
2. sensor.reset()       初始化摄像头 (QVGA RGB565)
3. find_bear(预热)       空跑一次推理, 提前分配 tensor arena
4. init_uart()          初始化串口通信
5. RobotState(40, −12)  初始化位姿状态
6. 初始化计数器           tennis/earthbag/bear_count = TOTAL 值
```

> **关键**: 熊模型必须在传感器初始化后、其他任何步骤前完成预热推理。否则后续步骤的堆碎片会导致 `tf.detect` 卡死。

## 5. 计数系统

三个计数器 `tennis_count` / `earthbag_count` / `bear_count` 控制循环生命周期：

- **初始化**: 等于对应的 `*_TOTAL` 常量
- **每次成功检测+推送**: 对应计数 `−1`
- **计数为 0 时**: 跳过该步骤
- **全部为 0**: 触发回家序列 → 信标重定位 → 重置计数 → 重新开始

```
判断全部清零的位置: step 3 末尾 (核心扫描循环结束后立即检查)
各步骤头部也检查自身计数, 计数=0 则跳过该步骤
```

## 6. 步骤详解

### 6.1 核心扫描循环 (Steps 1–3)

steps 1-3 构成一个扫描循环，依次处理网球、沙包、小熊。每个步骤的共性：

- **计数为 0** → 跳过（不移动、不扫描）
- **找到物体** → 偏移 → 推送至边缘 → 计数 −1
- **未找到** → 记录兜底坐标，继续下一步

三个步骤完成后, step 3 末尾立即检查计数, 决定回家还是启动下一轮核心扫描。

---

#### 步骤 1 — 网球扫描

```
┌─ tennis_count==0? ──YES──→ 跳过, step=2
│
├─ 确保朝向 0°
├─ 移动到 (110, 70)
├─ 沿 X 扫描: 110 → 210, 检测 _detect_tennis
│
├─ 找到?
│   ├─ YES → _push_to_edge(offset_wx=+10)
│   │         绑定当前朝向 (0°) → 推到 FIELD_HEIGHT (y=240)
│   │         tennis_count -= 1
│   └─ NO  → cmd_x = 210
│
└─ step = 2
```

**传感器模式**: RGB565 QVGA

**推送方向**: 朝向 `0°` → 推到场地顶部边界 `y=240`

#### 步骤 2 — 沙包扫描 (含 X 方向补扫)

```
┌─ earthbag_count==0? ──YES──→ 跳过, step=3
│
├─ 旋转至 −90° (X→)
├─ 移动到 (cmd_x, 170)
├─ 切换到 GRAYSCALE QQVGA
├─ 沿 Y 扫描: 170 → 70, 检测 _detect_earthbag
│
├─ Y扫找到?
│   ├─ YES → _push_to_edge(offset_wx=−10)
│   │         朝向 −90° → 推到 FIELD_WIDTH (x=320)
|   |         后退 10cm
│   │         earthbag_count -= 1
│   │
│   └─ NO  → ── X向补扫 ──
│              ├─ 移动到 (110, 70)
│              ├─ 旋转至 0°
│              ├─ 沿 X 扫描: 110 → 210
│              │
│              ├─ 找到?
│              │   ├─ YES → 偏移 X+10 → 移动到 Y=170
│              │   │        旋转 −90° → 推到 x=320
│              │   │        后退 10cm
│              │   │        earthbag_count -= 1
│              │   └─ NO  → cmd = (210, 70)
│
├─ 切回 RGB565 QQVGA
└─ step = 3
```

**传感器模式切换**: GRAYSCALE (沙包检测) → RGB565 (返回后)

#### 步骤 3 — 小熊扫描

```
┌─ bear_count==0? ──YES──→ do_scan=False
├─ bear_available==False? ──YES──→ do_scan=False
│
├─ do_scan==True?
│   ├─ 旋转至 0°
│   ├─ 移动到 (210, 70)
│   ├─ sensor.reset() RGB565 QVGA  ← 关键: 必须 reset
│   │                                (第2步 GRAYSCALE 残留需清理)
│   ├─ gc.collect()
│   ├─ 沿 Y 扫描: 70 → 170, 检测 scan_bear_direct()
│   │   (直接调用 find_bear, 不通过 _detect_bear 包装)
│   │
│   ├─ 找到?
│   │   ├─ YES → _push_bear_to_left_edge(offset_y=+5)
│   │   │         专用推送: 固定推向 x=0 (左边界)
│   │   │         bear_count -= 1
│   │   └─ NO  → ── X向补扫 ──
│              ├─ 移动到 (210, 70)
│              ├─ 旋转至 180°
│              ├─ 沿 X 扫描: 210 → 110
│              │
│              ├─ 找到?
│              │   ├─ YES → 偏移 X-10 → 移动到 Y=70
│              │   │        旋转 −90° → 推到 x=0
│              │   │        后退 10cm → 旋转 −90°
│              │   │        earthbag_count -= 1
│              │   └─ NO  → cmd = (110, 70)
│
├─ [计数检查]
│   ├─ 全零 → 回家序列 (返回起点 + 信标重定位 + 重置计数)
│   └─ 非全零 → 打印剩余 → step = 1
```

**传感器模式**: RGB565 **QVGA** (熊模型需要 QVGA 分辨率)

> **为什么必须 `sensor.reset()`**: 步骤 2 使用了 GRAYSCALE/QQVGA 模式。仅靠 `set_framesize` / `set_pixformat` 切换回 QVGA 时，帧缓冲重建不干净，`tf.detect` 会在逐像素读取阶段卡死。`reset()` 强制重建帧缓冲，确保与开机预热时的路径一致。

> **为什么用 `scan_bear_direct` 而不用 `_detect_bear`**: 在 OpenART mini 上，通过 `detect_fn` 回调包装后调用 `find_bear` 会使 `tf.detect` 卡死。直接调用可避免此问题。

### 6.2 回家序列

当 `tennis_count==0 and earthbag_count==0 and bear_count==0` 时触发：

```
1. send_move_to(110, 100)     中间过渡点
2. send_move_to(25, −25)      返回起点
3. sensor.snapshot()
4. localize_from_beacon()     AprilTag PnP 重定位
   检测 Tag#0 (110,240) 和 Tag#1 (210,240)
   根据 heading 计算世界坐标偏移
   同步下位机位置
5. 重置三个计数器为 TOTAL 值
6. step = 1 (开始新的一轮)
```

## 7. 推送机制

### 7.1 `_push_to_edge` (通用推送)

```
输入: (cur_x, cur_y), offset_wx, offset_wy
步骤:
  1. send_move_delta(offset_wx, offset_wy, speed=40)  横向偏移
  2. 根据 state.heading 决定边缘目标:
       0°   → (cur_x, FIELD_HEIGHT)   y=240
     −90°   → (FIELD_WIDTH, cur_y)    x=320
      90°   → (0, cur_y)              x=0
  3. send_move_to 推到边缘
```

### 7.2 `_push_bear_to_left_edge` (小熊专用)

```
输入: (cur_x, cur_y), offset_y
步骤:
  1. send_move_delta(0, offset_y)    Y 方向偏移
  2. query_position() 获取真实位置
  3. send_move_to(0, cur_y)          推到左边界 x=0
```

> 小熊专用推送不依赖 `state.heading`，因为小熊的推送方向固定为 X− (左边界)。

## 8. 扫描机制

### 8.1 `scan_until_centered` (通用扫描)

```
send_move_to(target_x, target_y)     启动连续移动
for frm in range(timeout):
    img = snapshot()
    r = detect_fn(img)
    if r 有效 & 横向居中:
        send_speed(0, 0)             急停
        query_position()             查下位机真实位置
        return (cx, cy, px, ax, ay, img)

超时: send_speed(0,0), return None
```

居中判定：`|cx − img.width/2| < CENTER_TOLERANCE_PX` (30px)

### 8.2 `scan_bear_direct` (小熊专用扫描)

与 `scan_until_centered` 逻辑相同，但直接调用 `find_bear(img)` 而非通过回调函数，避免 OpenART mini 上回调包装导致 `tf.detect` 卡死。

## 9. 传感器模式切换

| 步骤 | 模式 | 分辨率 | 原因 |
|------|------|--------|------|
| 初始化 | RGB565 | QVGA | 通用 |
| Step 1 | RGB565 | QVGA | 网球使用彩色检测 |
| Step 2 主扫 | **GRAYSCALE** | QQVGA | 沙包用灰度检测 |
| Step 2 结束 | RGB565 | QQVGA | 恢复彩色 |
| Step 3 | RGB565 | **QVGA** | 熊模型需要 QVGA (YOLO 输入 112×112) |


> Step 3 必须 `sensor.reset()` 而非仅 `set_framesize/set_pixformat`，否则帧缓冲不干净导致 `tf.detect` 卡死。

## 10. 信标 PnP 重定位 (`localize_from_beacon`)

仅在**回家序列**中触发。

```
检测 AprilTag → 获取 tag_id
查找 BEACON_WORLD_POSITIONS[tag_id] → (beacon_wx, beacon_wy)
根据 state.heading 将相机坐标系下的 (tx, tz) 转换为世界偏移 (wdx, wdy):
  heading=0°:   wdx=tx,   wdy=tz
  heading=−90°: wdx=tz,   wdy=−tx
  heading=90°:  wdx=−tz,  wdy=tx
  heading=180°: wdx=−tx,  wdy=−tz

小车世界坐标 = (beacon_wx − wdx, beacon_wy − wdy)
夹紧到 [0, FIELD_WIDTH] × [0, FIELD_HEIGHT]
send_set_position() 同步下位机
```

## 11. 通信协议

下位机通过 UART ASCII 帧通信：`@...#...!`

| 函数 | 指令 | 说明 |
|------|------|------|
| `send_rotate(angle)` | 旋转 | 相对角度 |
| `send_move_to(x, y)` | 移动到绝对坐标 | 连续移动 |
| `send_move_delta(dx, dy, speed)` | 相对位移 | |
| `send_set_position(x, y)` | 设置下位机坐标 | 不移动 |
| `send_speed(vx, vy)` | 设置速度 | `0,0` = 急停 |
| `query_position()` | 查询当前位置 | 返回 `(x, y)` |

## 14. 边界情况

| 情况 | 行为 |
|------|------|
| 某步骤计数=0 | 跳过该步骤，不移动 |
| Step 2 Y扫未找到 | 返回 (110,70)，旋转 0°，X 方向补扫 |
| Step 2 X补扫也未找到 | cmd=(210,70)，继续 step 3 |
| Step 3 Y扫未找到 | 返回 (210,170)，旋转 0°，X 方向补扫 |
| Step 3 X补扫也未找到 | cmd=(110,70)，继续 step 1 |
| 熊模型不可用 | 跳过 step 3，不阻塞流程 |
| 扫描超时 (300帧) | 急停，返回 None，跳过该物体 |
| 查询位置失败 | 用终点坐标兜底 |
| 信标未检测到 | 使用命令坐标继续，不影响主循环 |
| 所有物体都未找到 | 无限循环重试 (需人工介入或调大 SCAN_TIMEOUT) |
| 某物体已清零 | 自动跳过该步骤, 循环继续 (只扫描仍有剩余物体的步骤) |
